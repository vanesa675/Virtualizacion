Desventajas


  • Coste del software de virtualización: Algunos programas de virtualización pueden ser costosos, especialmente para empresas pequeñas o medianas.
  • Complejidad: La configuración y administración de la virtualización puede ser compleja, especialmente para usuarios sin experiencia. Se requiere personal con conocimientos específicos para gestionar la infraestructura virtualizada.
  • Rendimiento: La virtualización puede afectar el rendimiento del equipo físico, especialmente si se ejecutan muchas VM en un mismo equipo.
  • Dependencia del hipervisor: La virtualización depende del software de hipervisor para funcionar. Si el hipervisor falla, todas las VM se verán afectadas.
  • Riesgo de seguridad: La virtualización puede aumentar la superficie de ataque, ya que hay más software y sistemas operativos que pueden ser vulnerables a ataques.
  • Vulnerabilidades del software de virtualización: El software de virtualización puede tener vulnerabilidades que pueden ser explotadas por los atacantes.
  • Posible incompatibilidad: Es posible que algunos dispositivos o aplicaciones no sean compatibles con la virtualización.
  • Dificultad para la resolución de problemas: Los problemas en un entorno virtualizado pueden ser difíciles de diagnosticar y solucionar.
  • Necesidad de formación: El personal necesita formación para poder administrar la infraestructura virtualizada de forma eficaz.
  • Dependencia de la energía: La virtualización depende de un suministro de energía constante. Si hay un corte de energía, todas las VM se apagarán.
  • Mayor complejidad de la red: La virtualización puede aumentar la complejidad de la red, ya que se necesitan redes virtuales para conectar las VM.
  • Mayor consumo de energía: La virtualización puede aumentar el consumo de energía, ya que se necesitan más recursos para ejecutar varias VM en un mismo equipo.
  • Mayor consumo de almacenamiento: Las VM necesitan espacio de almacenamiento para el sistema operativo, las aplicaciones y los datos.
